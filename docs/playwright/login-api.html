Objetivo

Aprender a localizar elementos correctamente y documentar los selectores útiles para pruebas automatizadas con Playwright.

Selectores

  • getByRole('button', { name: 'Ingresar' })

    Propósito: Selecciona el botón por su rol accesible y texto visible. Recomendado por su estabilidad y alineación con la experiencia del usuario.

  • page.locator('.button').first()

    Propósito: Selecciona el primer elemento que coincide con la clase .button. Útil cuando existen múltiples coincidencias.

  • page.locator()

    Propósito: Selector genérico que acepta CSS o XPath. Preferir selectores más específicos cuando sea posible.

Estrategia Playwright

  • Priorizar `getByRole()` y selectores basados en accesibilidad cuando estén disponibles.
  • Evitar XPath absoluto; preferir `data-testid`, roles o selectores CSS estables.
  • Ejemplo práctico: 
    await page.getByRole('button', { name: 'Ingresar' }).click();
  • Si hay más de un elemento coincidente, usar `.first()`, `nth()` o definir selectores más específicos para evitar errores de "Strict mode violation".

Observaciones QA

  • Verificar que los selectores resistan cambios de UI y no dependan de texto volátil.
  • Agregar `data-testid` en componentes críticos para tests más robustos cuando sea posible.
  • Comprobar accesibilidad (roles, labels) para poder utilizar `getByRole` de forma fiable.
  • Documentar el propósito de cada selector en el repositorio para facilitar mantenimiento por QA Automation.
